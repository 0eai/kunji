kunji issuer demo

A verified-credentials issuer: it signs SD-JWT VCs about a holder, publishes its keys, and serves a StatusList for revocation. A relying party verifies a presentation locally against these keys — kunji is not in the path.

  • /.well-known/kunji-issuer.json — the signing keys
  • POST /issue { holderJwk, vct?, claims? } → an SD-JWT VC
  • GET /status/1?idx=N → { valid } (revocation)

Drive it headlessly from the RP demo: cd ../kunji-node-demo && ISSUER=<this origin> npm run wallet -- --vc. See docs/verified-credentials.md.
