kunji agent demo

A relying party that accepts both human logins and agent logins. No Firebase, no framework — just Node + an Ed25519 signature.

An authorized agent can sign in to this same RP without a fresh approval — see the README (npm run agent or the kunji-mcp bridge). It runs against the backend; watch the terminal, or GET /kunji/status?sessionId=….
